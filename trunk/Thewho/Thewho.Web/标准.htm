命名规则

文件夹和网页文件,js文件,CSS文件命名: 全小写,如果有第二个单词,用下划线隔开,英文或者英文缩写, 如:person, person_doc.aspx,person_style_main.js
图片命名规则: 全小写,如果有第二个单词,用下划线隔开,英文或者英文缩写,允许英文开头,以数字区分结尾,如:nav_bg.gif, uc1.gif,uc2.gif
CSS命名规则: 全小写,如果有第二个单词,用下划线隔开,英文或者英文缩写,允许英文开头,以数字区分结尾,如:class="button_login",id="button_reg"
对象id命名规则: 全小写,如果有第二个单词,用下划线隔开,英文或者英文缩写,如:btn_search

自定义变量命名规则:骆驼命名法,如 int i = 0; string strSql = "";
方法命名规则:首字母大写,如果有第二个单词,不用隔开,第二个首字母也大写,以此类推, 不允许出现数字,如: GviewBind()


数据库表,字段命名法:首字母大写,如果有第二个单词,不用隔开,第二个首字母也大写,以此类推, 不允许出现数字,如: PersonInfo表, NickName字段
模型层类文件命名法:首字母大写,如果有第二个单词,不用隔开,第二个首字母也大写,以此类推, 不允许出现数字,如: PersonInfo.cs (和数据库对应)

主要标题:h1 只有一个和title一致, 比如一篇文章的标题 突出当前最核心的内容到 ，。

当前列表h2 当前类别名称. 突出层次感 ，。

小标题h3 用于其他的分类.

关键字提示:strong 作用于单个突出的关键字 或者 关键句提示 加粗
关键字提示补充:em 少用.
只用来显示加粗 不涉及优化
